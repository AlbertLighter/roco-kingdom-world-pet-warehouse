# 洛克王国孵蛋概率计算器核心逻辑分析 (V2.8.1)

本文档总结了从 `docs/c.html` 提取的洛克王国精灵孵蛋（遗传）概率计算器的技术逻辑。

## 1. 基础数据定义

### 属性槽位 (SLOTS)
系统定义的六维属性顺序为：
`["生命", "物攻", "物防", "魔攻", "魔防", "速度"]`

### 性格数据 (NATURE_DATA)
包含所有性格及其对应的属性修正（加成属性 `plus` 和减益属性 `minus`）。
*   例如：`固执: {plus: "物攻", minus: "魔攻"}`，`平衡` 类性格（如实干）则无修正。

## 2. 核心计算算法 (`BreedCalculator`)

### A. 遗传权重 (Weights) 计算
每个属性在遗传时都有一个动态权重，由父母本的个体情况决定：
*   **基础权重**: 每个属性固定基础值为 `100`。
*   **父母基因加成**: 
    *   如果父本在该属性上有勾选（代表优秀个体），权重 +300。
    *   如果母本在该属性上有勾选，权重 +300。
*   **公式**: `Weight(attr) = 100 + 300 * (父本拥有? 1:0 + 母本拥有? 1:0)`

### B. 遗传条数概率 (k-Count Probs)
决定子代会遗传 1、2 还是 3 条属性的概率：
*   **非国王球模式**:
    *   1条权重 `w1 = 0` (固定)。
    *   2条权重 `w2 = 100 + (父本勾选2条? 300:0) + (母本勾选2条? 300:0)`。
    *   3条权重 `w3 = 100 + (父本勾选3条? 300:0) + (母本勾选3条? 300:0)`。
    *   **概率**: `P(k) = wk / (w1 + w2 + w3)`。
*   **国王球契约模式**: 
    *   固定遗传 3 条属性，即 `P(3) = 100%`。

### C. 组合抽样模型
计算特定属性组合（如“生命+速度”）的遗传概率，采用**无放回抽样 (Sampling without replacement)**：
1.  计算该组合所有可能的排列顺序。
2.  对于每种排列，概率 = `Π (目标属性权重 / 剩余可选属性总权重)`。
3.  将所有排列的概率相加。

## 3. 国王球契约 (KingBall) 特殊逻辑

当开启国王球契约时：
1.  **锁定属性**: 用户必须选择一个“正面性格属性”（`kingBallAttr`），其遗传概率强制为 `100%`。
2.  **剩余槽位**: 子代剩下的 2 个遗传槽位从其余 5 个属性中，根据上述权重逻辑进行无放回抽样。

## 4. 目标概率与模拟

### 完美继承概率 (`getTargetProb`)
计算子代**同时**包含用户选定的 1~3 个目标属性的概率。
*   如果不满足基本前提（如国王球未覆盖目标属性之一），概率直接为 0。

### 孵化模拟
计算准备 `n` 个蛋时，至少成功获得一个目标个体的概率：
*   **公式**: `P_at_least_one = 1 - (1 - 目标概率)^n`

## 5. 性格继承判定

存档系统会对性格进行逻辑分析：
*   **完美继承**: 子代性格与父母双方均一致。
*   **单方继承**: 子代性格仅与父本或母本一致。
*   **属性继承**: 判定子代的“加成属性”或“减益属性”是否分别遗传自父本或母本。
*   **全歪**: 子代性格修正与父母本均无关联。

## 6. 遗传基础规则

### A. 种族决定
*   **规则**: 繁殖生成的宠物种族由**母方**（雌性）决定。

### B. 性格遗传概率
子代的性格产生遵循以下分布逻辑：
*   **母方遗传**: 30% 概率直接继承母方性格。
*   **父方遗传**: 30% 概率直接继承父方性格。
*   **随机生成**: 40% 概率在全性格池中随机生成。
